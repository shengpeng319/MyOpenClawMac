# 研究与信息搜集方法论

## 核心原则

**多源验证 vs 单源依赖**

- ❌ **不要只从一个 APP/网站搜集信息**
- ✅ **先研究口碑好的信息源**
- ✅ **选出多个可靠源进行交叉验证**
- ✅ **综合总结，形成全面视角**

---

## 🔧 经验学习（2026-02-07）

### 多源验证的重要性
- **教训**：一开始只去知乎搜集新闻
- **改进**：扩展到虎嗅、36氪等多平台
- **结果**：形成更全面、更有深度的新闻汇总

## 信息搜集标准流程

### Step 1: 定义需求
- 明确要搜集什么类型的信息
- 确定需要几个维度的观点

### Step 2: 选择信息源
- 根据需求选择合适的新闻源
- 优先选择口碑好、专业度高的平台

### Step 3: 多源搜集
- 从每个源获取关键信息
- 记录不同视角的观点

### Step 4: 综合总结
- 提炼热点话题
- 分类整理
- 提供洞察

# 深度研究方法论 - OpenClaw 研究指南

> **核心理念**：不只是搜集信息，而是理解信息背后的逻辑、关联和洞察。

---

## 研究的基本原则

### 1. 深度 vs 广度
- **浅层研究**：搜集信息 → 简单汇总 → 表面结论
- **深度研究**：多源验证 → 关联分析 → 本质洞察

### 2. 批判性思维
- 质疑信息来源
- 区分事实与观点
- 识别偏见和立场
- 寻找因果关系

### 3. 系统性思维
- 看到事物的全貌
- 理解各部分关联
- 识别模式和趋势
- 预测未来走向

## 第一阶段：定义问题

### 1.1 明确研究目标

**好的问题** vs **差的问题**

| 差的问题 | 好的问题 |
|---------|---------|
| "AI是什么？" | "AI在医疗领域的应用前景和挑战是什么？" |
| "特斯拉怎么样？" | "特斯拉在中国市场的竞争优势和劣势分析" |
| "房价会涨吗？" | "2026年一线城市房价走势的影响因素" |

### 1.2 拆解问题

使用 **MECE 原则**（相互独立，完全穷尽）

示例：研究"为什么年轻人不愿意生孩子"

```
问题拆解：
├── 经济因素
│   ├── 房价压力
│   ├── 教育成本
│   └── 就业不稳定
├── 观念因素
│   ├── 个人主义兴起
│   ├── 婚姻观念变化
│   └── 生育认知改变
├── 制度因素
│   ├── 产假政策
│   ├── 托育服务
│   └── 社会保障
└── 环境因素
    ├── 育儿时间成本
    ├── 个人发展空间
    └── 代际关系
```

### 1.3 设定边界

- **时间边界**：研究范围（1年？5年？）
- **地域边界**：地理范围（全球？中国？）
- **数据边界**：信息来源限制
- **认知边界**：自身知识局限

## 第三阶段：深度分析

### 3.1 分析工具箱

#### 工具1：5 Whys 分析法
```
问题：为什么特斯拉销量下降？

Why 1: 销量下降
  └── 为什么？→ 用户转向竞争对手

Why 2: 转向竞争对手
  └── 为什么？→ 竞争对手价格更低

Why 3: 竞争对手价格低
  └── 为什么？→ 成本控制更好

Why 4: 成本控制更好
  └── 为什么？→ 本地化生产

Why 5: 本地化生产
  └── 本质原因：供应链本地化是核心竞争力
```

#### 工具2：波特五力分析（行业分析）
```
行业吸引力评估：

1. 现有竞争者强度
   ├── 竞争者数量：__
   ├── 行业集中度：__
   └── 价格竞争程度：__

2. 潜在进入者威胁
   ├── 壁垒高度：__
   ├── 规模经济要求：__
   └── 资本要求：__

3. 替代品威胁
   ├── 替代品数量：__
   ├── 替代品性价比：__
   └── 用户转换成本：__

4. 供应商议价能力
   ├── 供应商集中度：__
   ├── 替代供应来源：__
   └── 供应重要性：__

5. 客户议价能力
   ├── 客户集中度：__
   ├── 价格敏感度：__
   └── 差异化程度：__
```

### 3.2 关联分析方法

**知识图谱构建法**

```
识别实体：
- 主体：企业A、产品B、人物C
- 关系：投资、竞争、合作、监管
- 影响：市场、技术、政策

关联网络：
A ───投资──→ B（资本关联）
B ───竞争──→ C（市场关联）
C ───监管──→ D（政策关联）
D ───影响──→ A（反向关联）
```

### 3.3 数据分析方法

| 分析类型 | 方法 | 适用场景 |
|---------|------|----------|
| 描述分析 | 平均值、中位数、分布 | 现状描述 |
| 趋势分析 | 时间序列、环比同比 | 变化规律 |
| 相关分析 | 相关系数、回归分析 | 因素关联 |
| 对比分析 | A/B测试、标杆对比 | 差异识别 |
| 预测分析 | 回归预测、时间序列 | 未来趋势 |

## 第四阶段：洞察提炼

### 4.1 从数据到洞察

```
数据 → 信息 → 知识 → 洞察

数据：2025年新能源汽车销量增长30%
  ↓
信息：比亚迪、蔚来、小鹏销量排名变化
  ↓
知识：价格战导致市场份额重新分配
  ↓
洞察：2026年价格战仍将持续，但差异化竞争将成为新战场
```

### 4.2 洞察检验清单

```markdown

### 逻辑性
- [ ] 因果关系清晰
- [ ] 逻辑链条完整

### 证据性
- [ ] 有数据支撑
- [ ] 有案例验证

### 实用性
- [ ] 可指导行动

### 局限性
- [ ] 边界条件已说明
```

|---------|------|----------|
| 事实洞察 | 客观存在的规律 | "数据显示..." |
| 原因洞察 | 现象背后的成因 | "根本原因是..." |
| 趋势洞察 | 未来走向的判断 | "预计将..." |
| 机会洞察 | 潜在的可能性 | "存在...机会 |
| 风险洞察 | 潜在的危险信号 | "需警惕...风险 |
